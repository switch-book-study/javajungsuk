## 프로세스와 쓰레드
  - 프로세스는 실행중인 프로그램이다.
  - 프로그램을 실행하면 OS로 부터 실행에 필요한 자원(메모리)을 할당받아 프로세스가 된다.
  ```
  싱글쓰레드 프로세스 = 자원 + 쓰레드
  멀티쓰레드 프로세스 = 자원 + 쓰레드 + 쓰레드 ...
  ```
  ```
  멀티쓰레딩의 장점
    - cpu 사용률을 향상시킨다.
    - 자원을 보다 효율적으로 사용할 수 있다.
    - 사용자에 대한 응답성이 향상된다.
    - 작업이 분리되어 코드가 간결해진다.
  ```
  ```
  멀티쓰레딩의 단점
    여러 쓰레드가 같은 프로세스 내에서 자원을 공유하면서 작업하기 때문에
    동기화문제(synchronization), 교착상태(deadlock) 문제를 고려하여 프로그래밍 해야한다.
  ```
